# TUN 数据包打印功能说明

## 功能介绍

现在 xTune 可以打印 TUN 设备接收到的所有数据包内容，包括：
- 原始十六进制数据转储
- 解析后的数据包信息
- 源/目标地址和端口
- 协议类型

## 打印输出格式

### 1. 十六进制转储

```
[数据包 #1] 接收到 60 字节:
  0000  45 00 00 3c 1c 46 40 00 40 06 00 00 c0 a8 01 01  │E...<.F@.@.......│
  0010  c0 a8 01 02 00 50 d8 5f 00 00 00 00 00 00 00 00  │.....P._........│
  0020  50 02 20 00 00 00 00 00 00 00 00 00 00 00 00 00  │P. .............│
  0030  00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00  │................│
  
  ✓ 解析成功: 192.168.1.1 -> 192.168.1.2 (协议: Tcp)
    端口: 192.168.1.1:80 -> 192.168.1.2:55391
```

### 2. 数据包信息

每个收到的数据包会显示：
- 数据包序号
- 数据包大小
- 源/目标 IP 地址
- 协议类型（TCP/UDP/Other）
- 源/目标端口（如果有）

## 打印策略

为了避免输出过多，程序采用以下打印策略：

### 前 5 个数据包
打印完整的十六进制转储和解析结果

### 之后的数据包
- 每 100 个数据包打印一次统计信息
- 每 100 个数据包打印一次十六进制转储（可选）

### 日志级别

在 Linux 上运行时：
```bash
# 查看完整日志（包括所有数据包信息）
sudo ./target/debug/xtune

# 或查看日志文件
tail -f xtune.log
```

## 十六进制转储说明

每行格式：
```
OFFSET  HEX DATA (16 bytes)                     │ ASCII 表示 │
0000    45 00 00 3c 1c 46 40 00 40 06 b2 51    │E..<.F@.@..Q│
```

- **OFFSET**: 数据包中的偏移位置（十六进制）
- **HEX DATA**: 原始字节数据（十六进制）
- **ASCII**: 可打印的字符（不可打印显示为 `.`）

## IPv4 头部结构解读

```
字节位置     十六进制        说明
0000         45             版本(4) + IHL(5)
0001-0002    00 3c          总长度: 60 字节
0003-0004    1c 46          标识
0005-0006    40 00          标志和分片偏移
0007         40             生存时间 (TTL)
0008         06             协议 (6=TCP, 17=UDP)
0009-000a    b2 51          校验和
000b-000e    c0 a8 01 01    源 IP: 192.168.1.1
000f-0012    c0 a8 01 02    目标 IP: 192.168.1.2
```

## 配置打印行为

可以通过修改以下代码来改变打印策略：

```rust
// src/main.rs 中的代码
// 打印前 N 个数据包
if packet_count <= 5 {
    // 打印十六进制转储
}

// 每 M 个数据包打印一次
if packet_count % 100 == 0 {
    // 打印统计信息
}
```

### 仅打印前 N 个包

```rust
if packet_count <= 10 {  // 改为 10，则仅打印前 10 个包
    // 打印十六进制转储
}
```

### 打印所有包

```rust
// 移除条件判断，始终打印所有包
log_message(&format!("\n[数据包 #{}] 接收到 {} 字节:", packet_count, n));
let hex_dump = format_hex_dump(&buf[..n], 8);
```

### 只打印统计，不打印详细信息

```rust
// 注释掉十六进制打印部分
// if packet_count <= 5 {
//     log_message(&format!("\n[数据包 #{}]...", packet_count, n));
// }
```

## 性能影响

- 打印前 5 个数据包：几乎无性能影响
- 打印所有数据包：可能降低 20-30% 的性能
- 打印到文件 vs 控制台：文件性能更好

## 运行方式

### 在 Linux 上运行（需要 root 权限）

```bash
# 编译
cargo build --release

# 以 root 身份运行
sudo ./target/release/xtune

# 或在后台运行，并查看日志
sudo ./target/release/xtune > xtune.log 2>&1 &
tail -f xtune.log
```

### 在演示模式下运行（不需要权限）

```bash
# 不需要 root
./target/debug/xtune
```

在演示模式下不会接收实际的 TUN 数据包，但所有代码都被编译并准备就绪。

## 故障排除

### 看不到任何数据包

1. **检查 TUN 设备权限**
   ```bash
   # 需要 root 权限
   sudo ./target/release/xtune
   ```

2. **检查网络活动**
   ```bash
   # 打开另一个终端，产生网络流量
   ping 8.8.8.8
   curl https://example.com
   ```

3. **查看日志文件**
   ```bash
   cat xtune.log
   ```

### 输出过多

可以修改代码来减少输出：
```rust
// 只打印前 2 个包
if packet_count <= 2 {
    // 打印详细信息
}
```
